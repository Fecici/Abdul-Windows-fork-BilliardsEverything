# Bug Register

Central register for confirmed and suspected bugs found during the Abdul source audit.

The durable tracker is now `tracker/issues.csv` plus one-line JSON in `tracker/issues.jsonl`. Candidate fixes live under `tracker/candidate-fixes/`. This Markdown file is a human-readable snapshot; do not treat it as more current than the structured tracker.

Status values:

- `open`: likely bug, not fixed.
- `confirmed`: source evidence proves the defect or serious risk.
- `fixed`: source fix has been applied; verification may still be pending.
- `needs repro`: plausible bug, but user-visible behavior still needs reproduction.
- `not a bug`: investigated and rejected.
- `deferred`: real issue, intentionally postponed.
